# GraphQL Instrumentation

The declarative configuration keys below are nested under the `graphql` instrumentation key (see
[declarative configuration](https://opentelemetry.io/docs/languages/java/configuration/)).

## Common settings (all GraphQL versions)

| System property                                                    | Declarative key                       | Type    | Default | Description                                                                                                                                                                                                          |
| ------------------------------------------------------------------ | ------------------------------------- | ------- | ------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `otel.instrumentation.graphql.query-sanitization.enabled`          | `query_sanitization.enabled`          | Boolean | `true`  | Whether to remove sensitive information from query source that is added as span attribute.                                                                                                                           |
| `otel.instrumentation.graphql.operation-name-in-span-name.enabled` | `operation_name_in_span_name.enabled` | Boolean | `false` | Whether GraphQL operation name is added to the span name. **WARNING**: GraphQL operation name is provided by the client and can have high cardinality. Use only when the server is not exposed to malicious clients. |
| `otel.instrumentation.graphql.capture-query`                       | `capture_query`                       | Boolean | `true`  | Whether to capture the query in `graphql.document` span attribute.                                                                                                                                                   |

## Additional settings for GraphQL 20 and higher

| System property                                                        | Declarative key                           | Type    | Default | Description                                                                                                                                                                                                                                         |
| ---------------------------------------------------------------------- | ----------------------------------------- | ------- | ------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `otel.instrumentation.graphql.data-fetcher.enabled`                    | `data_fetcher.enabled`                    | Boolean | `false` | Whether to create spans for data fetchers.                                                                                                                                                                                                          |
| `otel.instrumentation.graphql.trivial-data-fetcher.enabled`            | `trivial_data_fetcher.enabled`            | Boolean | `false` | Whether to create spans for trivial data fetchers. A trivial data fetcher is one that simply maps data from an object to a field.                                                                                                                   |
| `otel.instrumentation.graphql.operation-span.enabled`                  | `operation_span.enabled`                  | Boolean | `true`  | Whether to create the GraphQL operation span. When disabled, no `GraphQL Operation` span is created; data fetcher spans, if enabled, are unaffected.                                                                                                |
| `otel.instrumentation.graphql.add-attributes-to-local-root-span`       | `add_attributes_to_local_root_span`       | Boolean | `false` | Whether to add GraphQL attributes (`graphql.operation.name`, `graphql.operation.type`, `graphql.document`) and exception events to the local root span (typically the enclosing HTTP server span), in addition to or instead of the operation span. |
| `otel.instrumentation.graphql.promote-error-status-to-local-root-span` | `promote_error_status_to_local_root_span` | Boolean | `false` | Whether to set the local root span status to `ERROR` when the GraphQL result contains any errors. **WARNING**: This marks the enclosing (e.g. server) span as errored for any GraphQL error, including partial errors on an HTTP 200 response.      |
